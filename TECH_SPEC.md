# 코드 낙하 (Code Drop) — 기술 명세

> 작성: 2026-07-08 (Fable 5) · 짝 문서: `GAME_DESIGN.md`(규칙·수치의 원본 — 충돌 시 그쪽이 우선)
> **구현자(Sonnet) 준수 사항**: 이 두 문서에 없는 규칙·수치를 임의로 발명하지 말 것. 애매하면 구현을 멈추고 질문으로 남길 것.

---

## 1. 뻘짓 방지 조항 (최우선)

1. **프레임워크·라이브러리·빌드 도구·npm 금지.** 바닐라 HTML/CSS/JS. 파일을 더블클릭(file://)해서 바로 실행돼야 한다.
2. **웹폰트·CDN·외부 요청 0건.** 오프라인 교실이 기본 환경. 폰트는 시스템 스택(§6), 효과음은 WebAudio 합성(음원 파일 금지).
3. **canvas 금지 — 블록은 DOM 요소로 그린다.** 근거: 동시 블록 ≤ 3개라 성능 무의미, 텍스트 렌더·글자별 스타일(정타 채움)·접근성·반응형 모두 DOM이 압도적으로 쉽다. canvas를 쓰는 순간 글자 채움 표현이 대공사가 된다.
4. **GAME_DESIGN의 수치(속도·점수·등급표·파라미터)를 바꾸지 말 것.** 전부 §5의 상수 객체 한 곳에 두고 참조만 한다.
5. 입력은 **keydown 기반**(§4). `<input>`/`<textarea>`를 화면에 두지 말 것 — IME·포커스 문제의 근원.
6. 결과 화면의 학번·이름은 **localStorage에 저장하지 않는다**(개인정보 — 세션 메모리만). 기록(점수)은 이름 없이 저장.

## 2. 파일 구조 (2파일 배포)

```
code-drop/
├─ index.html    앱 전체 (CSS <style> + JS <script> 내장, ~1500줄 예상)
└─ snippets.js   스니펫 데이터 전용 (교사가 이 파일만 수정해 문제 교체)
```

- `snippets.js`는 `window.CODE_DROP_SNIPPETS = [...]` 전역 하나만 선언하는 classic script.
- index.html은 로드 실패(파일 누락) 시 **내장 기본 풀 15개로 폴백**하고 시작 화면에 "기본 문제로 실행 중" 배지를 띄운다 — 파일 하나만 복사한 학생 PC에서도 죽지 않게.
- 배포 = 폴더째 복사. 그 외 자산 없음.

## 3. 스니펫 데이터 스키마 (`snippets.js`)

```json
{
  "id": "l1-01",           // "l{level}-{번호}" 관례, 고유
  "level": 1,               // 0=비기너 풀, 1=초급 풀, 2=중급 풀, 3=고급 풀
  "category": "output",    // output|variable|operator|cast|builtin|condition|loop|input|list|string|function|dict|comprehension|slice|exception|module|tuple
  "code": "print(\"hello\")" // 판정 대상 문자열 그대로
}
```

- 로드 시 **유효성 필터**(통과 못 한 항목은 콘솔 경고 후 제외): ① `code`가 1~34자 ② ASCII 인쇄 가능 문자만(0x20~0x7E) ③ 개행·탭 없음 ④ 선행 공백 없음 ⑤ id 중복 없음.
- 초기 데이터: GAME_DESIGN §5의 100개를 그대로 입력한다(임의 수정 금지).
- ⚠️ **v1.1 패치(2026-07-10)**: 위험 경고(.block.danger, frame 루프에서 floorY*0.72 판정) · 타건음 콤보 피치(Audio2.key(combo)) · HUD 레벨 진행 도트(Hud.update) — 규칙 원본은 GAME_DESIGN §9 v1.1 항목.
- ⚠️ **확정된 변경(2026-07-10)**: ① 난이도 **비기너** 추가(level 0 풀, GAME_DESIGN §4.1·§5.0 — 유효성 검사도 level 0 허용, localStorage 최고기록 키 `beginner` 추가). ② 페이싱 수정 — `spawnFactor` 0.75→**0.6**, 화면에 블록 0개면 스폰 대기를 **0.6초로 클램프**(`CONFIG.emptyRespawn`). 난이도 선택 숫자키는 1=비기너 2=초급 3=중급 4=고급.
- 난이도별 추첨: GAME_DESIGN §4.1의 혼합비(예: 중급 = Lv1 30% + Lv2 70%)로 가중 무작위, 동일 스니펫은 최근 5블록 내 재등장 금지(최근 큐로 구현).

## 4. 입력 엔진 (가장 실수하기 쉬운 부분)

- `document`에 keydown 리스너 1개. PLAYING 상태에서만 판정.
- **판정 키**: `e.key.length === 1`인 인쇄 가능 키만 코드 문자와 비교. Shift 조합은 e.key가 이미 결과 문자(`"`, `(`, `:` 등)이므로 별도 처리 불필요.
- **무시(판정·통계 제외)**: Backspace, Tab, 화살표, 한/영, CapsLock 등 비인쇄 키. 단 ESC=일시정지, 결과·메뉴 화면의 Enter/R은 화면 전환용.
  - ⚠️ **구현 시 확정된 편차(2026-07-09)**: GAME_DESIGN §2·본 문서 초안은 일시정지를 "ESC/P"로 적었으나, **PLAYING 중 `p`/`P`는 코드 글자(`print` 등 최빈 문자)라 반드시 판정 대상이어야 함** → `P`를 일시정지 키에서 제외. **PLAYING 일시정지는 ESC 전용**, PAUSED 화면에서만 P/Enter/ESC로 재개. 헤드리스 프로브에서 `p` 미판정 버그로 발견·수정.
- **preventDefault 필수**: Space(스크롤), `'`·`/`(Firefox quick find), Tab(포커스 이탈) — PLAYING 중에만.
- **IME 감지**: `e.isComposing === true` 또는 `e.keyCode === 229`면 판정하지 않고 HUD를 `[한글 ⚠]` 상태로 전환 + 경고 배너. 이후 정상 keydown이 오면 `[영문 A]`로 복귀. (compositionstart/end 이벤트도 병행 청취하면 더 견고.)
- **타겟팅 규칙** (GAME_DESIGN §3.2): 타겟 없음 상태에서 입력 → 첫 글자 일치 블록 중 y가 가장 큰(바닥에 가까운) 것을 타겟으로. 일치 없으면 오타 집계만. 타겟은 처치 또는 낙하 소멸까지 고정.
- 통계 집계: 정타 수 / 오타 수(문자별 Map — 결과 화면 TOP 3용) / 오타 난 스니펫의 category Map.

## 5. 모듈 구조 (단일 script 내 논리 분할)

| 모듈(객체) | 책임 | 상태 |
|---|---|---|
| `CONFIG` | GAME_DESIGN §4의 모든 수치 상수(난이도 표·곡선 계수·점수식·등급표) | 불변 |
| `ScreenManager` | 상태 머신: TITLE→DIFFICULTY→COUNTDOWN→PLAYING⇄PAUSED→RESULT. 화면 = `<section>` 5개 show/hide, 전환 시 포커스 이동(§7-3) | currentScreen |
| `Game` | 판당 상태(생명·점수·콤보·레벨·처치 수·통계), 시작/종료/레벨업 판단 | 판당 리셋 |
| `Spawner` | 스폰 타이머, 혼합비 추첨, 레인 배정, 동시 상한 관리 | 최근 큐 |
| `Block` | 블록 1개: DOM 노드, 코드, 진행 인덱스, y 위치, 낙하 시간. 글자 span 분할 렌더(정타 채움용) | 배열 ≤3 |
| `InputEngine` | §4 전체. Game·Block에 정타/오타/처치 이벤트 전달 | 타겟 참조 |
| `Loop` | 단일 requestAnimationFrame. dt(초) 기반 y 갱신: `y += (필드높이/T_fall) × dt`. PAUSED·백그라운드 시 dt 버림(복귀 프레임 dt=0 처리 — 순간이동 방지) | rafId |
| `Hud` | 상단 바 갱신(변경된 값만 텍스트 교체 — 매 프레임 전체 재렌더 금지) | — |
| `Result` | 결과 계산(CPM = 정타 ÷ 플레이시간(분), 등급표), 약점 TOP3, 요약 복사(클립보드 + execCommand 폴백), 최고 기록 갱신 | — |
| `Audio` | WebAudio 신스 4음(타건 1kHz 10ms / 처치 상승 2음 / 오타 220Hz 60ms / 낙하 하강음). 첫 사용자 입력 후 AudioContext 생성(자동재생 정책) | muted |
| `Storage` | localStorage 래퍼. 실패(시크릿 모드) 시 조용히 무시 | — |

### localStorage 스키마

```json
// key: "codedrop_best"
{ "easy":   { "score": 12480, "acc": 94.2, "cpm": 182, "grade": "A", "hardcore": false, "date": "2026-07-08" },
  "normal": { ... }, "hard": { ... } }
// key: "codedrop_settings"
{ "sound": true, "fontSize": 2, "hardcore": false }   // fontSize: 1|2|3 = 16|19|22px
```

## 9. 성능·크기 목표

- 단일 rAF 루프, 프레임당 DOM 쓰기 = 블록 transform ≤ 3회 + 변경된 HUD 텍스트만. 60fps 유지(저사양 교실 PC 전제).
- 총 용량 목표 ≤ 120KB (2파일 합, 압축 전). 이미지·음원 0개.
