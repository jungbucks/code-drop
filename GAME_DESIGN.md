## 1. 핵심 루프 (30초 요약)

화면 상단에서 **Python 코드 한 줄이 적힌 블록**이 떨어진다 → 플레이어가 그 코드를 **정확히 타이핑**하면 블록 소멸 + 점수 → 블록이 **바닥에 닿으면 생명 −1** → 생명 0이면 게임 오버 → 결과 화면(점수·정확도·CPM)에서 **결과 요약 복사(제출용)**.

게임의 학습 목표: ① 파이썬 문법 기호(`:` `==` `()` `[]` `""`)를 손에 익히기 ② 영문 코드 타이핑 유창성 ③ 스니펫을 반복 노출시켜 문법 형태 암기.

## 2. 상태 흐름도

```
[TITLE 시작 화면]
   │ "시작하기" (Enter/클릭)
   ▼
[DIFFICULTY 난이도 선택]  ← 비기너 / 초급 / 중급 / 고급 (+ 하드코어 토글, §6)
   │ 선택 (Enter/클릭)
   ▼
[COUNTDOWN 3·2·1]  (2.4초, 이 동안 첫 블록 예고 표시)
   ▼
[PLAYING] ──ESC/P──▶ [PAUSED] ──ESC/P/Enter──▶ [PLAYING]
   │                      │ "그만하기"
   │ 생명 0                ▼
   ▼                  [TITLE]
[RESULT 결과 화면]
   │ "다시 하기"(R/Enter) → [COUNTDOWN] (같은 난이도)
   │ "난이도 바꾸기" → [DIFFICULTY]
   └ "처음으로" → [TITLE]
```

- PAUSED 중 낙하·스폰·타이머 전부 정지. 일시정지 남용 방지 장치는 두지 않는다(학습용 — 언제든 쉬어도 됨).
- 탭이 백그라운드로 가면(visibilitychange) 자동 PAUSED.
- RESULT는 게임 오버뿐 아니라 **레벨 10 클리어(완주)** 시에도 진입한다(§5). 완주 시 결과 화면에 "완주 🎉" 배지.

## 3. 게임 규칙 상세

### 3.1 블록

- 블록 = 코드 한 줄이 적힌 알약형 카드. 등장 시 화면 상단 밖에서 슬라이드 인, 이후 **등속 낙하**.
- 가로 위치: 3개 레인(좌/중/우) 중 무작위, 단 **직전 블록과 다른 레인** 우선. 동시 블록이 겹쳐 보이는 것 방지.
- 같은 스니펫은 **한 판에서 최소 5블록 간격**을 두고 재등장 가능(무작위 체감 보장).

### 3.2 타이핑 판정 (핵심 메커니즘)

- **자동 타겟팅**: 입력창이 따로 없다. 키를 누르면 화면의 블록 중 **입력 접두사와 일치하는 블록**이 자동으로 타겟이 된다.
  - 첫 글자가 여러 블록과 일치하면 **바닥에 가장 가까운 블록** 선택.
  - 타겟이 정해지면 그 블록 처치(또는 오타 후에도 유지)까지 타겟 고정. 타겟 블록은 테두리 강조 + ▶ 마커(§8 접근성).
  - 첫 글자가 어떤 블록과도 불일치 → 오타 1회로 집계(타겟 미지정 상태 유지).
- **글자 판정**: 타겟 블록의 코드와 앞에서부터 1:1 비교. 맞은 글자는 블록 안에서 **채워진 스타일**로 변한다(진행도가 블록 위에 그대로 보임 — 별도 입력 표시줄 없음).
- 마지막 글자까지 맞추면 블록 소멸(0.18초 페이드+축소), 점수 지급.
- **공백·대소문자·기호 전부 정확 일치** 요구. 코드니까 당연 — 이것이 이 게임의 학습 포인트다.

### 3.3 오타 처리 — 권장안: **전진 차단형 (부분 인정)** ✅

| 방식 | 동작 | 판단 |
|---|---|---|
| **전진 차단형 (채택)** | 오타 키는 무시되고 커서가 안 움직임. 이미 맞춘 글자는 유지. 오타는 통계·콤보에만 반영 | ✅ 권장 |
| 즉시 리셋형 | 오타 1회에 그 블록 입력 전체 무효화 | 하드코어 모드 옵션으로만 (§6) |

**근거**: ① 학습용이다 — 긴 스니펫에서 즉시 리셋은 초·중학생의 좌절→이탈로 직결. ② 전진 차단형은 **백스페이스가 필요 없어** 조작이 "정타만 치면 됨" 하나로 단순해진다(백스페이스 키는 게임에서 무시). ③ 오타가 정확도·콤보에 즉시 반영되므로 긴장감은 유지된다. ④ 타이핑 게임 표준(ZType, Typing of the Dead)과 같은 모델이라 학생에게 직관적.

- 오타 피드백: 블록이 좌우 2px 진동(0.15초, reduced-motion 시 진동 대신 테두리 점멸 1회) + 커서 위치에 ✗ 아이콘 0.3초 표시 + (사운드 켠 경우) 낮은 톤 비프. **색상만으로 알리지 않는다**(§8).

### 3.4 생명과 낙하 실패

- 시작 생명 **5** (하트 5개). 블록이 바닥선에 닿으면: 생명 −1, 블록은 붉은 플래시 후 제거, 콤보 리셋, 해당 블록의 남은 글자 수만큼 "놓친 글자"로 집계(정확도에는 미반영 — 정확도는 순수 타건 기준, §7).
- 타겟 중이던 블록이 바닥에 닿으면 타겟 해제.
- 생명 0 → 즉시 RESULT.

### 3.5 콤보와 점수

- **콤보 = 오타 없이 연속으로 처치한 블록 수** (블록 단위). 오타 1회 또는 블록 낙하 실패 시 0으로 리셋.
- 점수 계산 (블록 처치 시 일괄 지급):
  - 기본점 = 블록 글자 수 × 10
  - 콤보 배수 = 1 + 0.1 × min(콤보, 10)  → 최대 ×2.0
  - 노미스 보너스 = 그 블록을 오타 0으로 클리어 시 +30
  - `획득 = round(기본점 × 콤보 배수) + 노미스 보너스`
- 콤보 5·10·15…마다 HUD 콤보 배지가 한 단계 커짐(과한 파티클 금지 — §9 톤).

## 4. 난이도 3종 + 레벨 곡선

### 4.1 난이도(모드)별 기본 파라미터

| 파라미터 | 비기너 | 초급 | 중급 | 고급 |
|---|---|---|---|---|
| 스니펫 풀 | Lv0 (§5.0) | Lv1 (§5.1) | Lv1 30% + Lv2 70% | Lv2 30% + Lv3 70% |
| 시작 낙하 시간 T₀ (스폰→바닥) | 20초 | 16초 | 12초 | 9초 |
| 낙하 시간 하한 T_min | 9초 | 7초 | 5.5초 | 4초 |
| 시작 동시 블록 상한 | 1개 | 1개 | 1개 | 2개 |
| 최대 동시 블록 상한 | 2개 (레벨 7+) | 2개 (레벨 6+) | 3개 (레벨 4+에서 2개, 8+에서 3개) | 3개 (레벨 5+) |
| 생명 | 5 | 5 | 5 | 4 |

> 비기너 추가(2026-07-10): 영타 경험이 거의 없는 학생용. 기호 없는 파이썬 낱말(§5.0)만 등장 — 코드 문장 이전에 "파이썬 어휘 + 영문 자판"부터 손에 익히는 단계.

### 4.2 레벨 곡선 (모든 난이도 공통 규칙)

- **레벨업 조건: 블록 8개 처치마다 +1** (레벨 1→10, 10이 마지막).
- 낙하 시간: `T(level) = max(T_min, T₀ × 0.92^(level−1))` — 매 레벨 8%씩 빨라지고 하한에서 멈춘다.
- 스폰 간격: `S(level) = T(level) × 0.6 / 동시상한` — 동시 상한이 늘면 스폰도 촘촘해진다. 단 **화면의 블록 수가 동시 상한에 도달해 있으면 스폰 보류**.
- **빈 화면 즉시 스폰** (2026-07-10 페이싱 수정): 화면에 블록이 하나도 없으면 남은 스폰 대기와 무관하게 **0.6초 안에 다음 블록 스폰**. (기존 계수 0.75 + 보류 없음 조합은 빨리 처치한 학생일수록 빈 화면을 오래 보는 문제 — 계수도 0.75→0.6으로 단축.)
- 레벨업 순간: 화면 상단에 "LEVEL N" 1초 표시(게임은 계속), 진행 중 블록 속도는 **다음 스폰부터** 적용(이미 떨어지는 블록의 속도는 바꾸지 않는다 — 급변 억울함 방지).
- **레벨 10에서 8개를 더 처치하면 완주** → RESULT(완주 배지).

### 4.3 체감 검증용 기준치 (구현 후 튜닝 확인용)

- 초급 레벨 1: 16초 안에 평균 14자 스니펫 → 분당 약 53타만 쳐도 여유. 초등 저학년 하한 커버.
- 고급 레벨 10: 4초 안에 평균 22자 → 분당 330타 상당. 최상위권 도전 구간.
- 구현 후 위 두 극단에서 "초급 1레벨이 지루하지 않은지 / 고급 10레벨이 이론상 가능한지"를 확인할 것.

## 6. 모드 옵션

- **하드코어 토글** (난이도 선택 화면의 체크박스, 기본 꺼짐): 오타 시 해당 블록 입력 전체 리셋(즉시 리셋형). 상위권 학생용 도전 모드. 결과 화면과 기록에 "하드코어" 표기.
- **🏅 오늘의 도전 토글** (v1.3, 2026-07-11 추가 — 난이도 선택 화면 체크박스 `C`키, 기본 꺼짐): 켜면 `날짜(yyyy-MM-dd)+난이도`를 시드로 **결정론적 난수(mulberry32)**를 써서 문제·레인 순서를 고정한다 → **같은 날·같은 난이도면 반 전원이 똑같은 문제 순서**를 받아 점수 비교(반 대항전)가 성립. 결과 화면·복사 텍스트에 "🏅오늘의 도전 {날짜}" 표기(교사 수합 식별용). 세션만 유지(비영속). "다시 하기"는 같은 시드 → 같은 문제(연습 반복). 근거: 시드는 콘텐츠 순서만 고정 — 타이핑 속도·레벨은 사람마다 달라도 등장하는 문제 N번째는 동일.
- **연습 모드는 만들지 않는다** — 초급 레벨 1이 그 역할(낙하 16초 = 사실상 무압박). 모드 수를 늘려 선택 마비를 만들지 말 것.

## 7. HUD와 결과 화면

### 7.1 게임 중 HUD (상단 고정 한 줄, 최소주의)

```
♥♥♥♥♥   점수 12,480   콤보 ×7   LV 3   [영문 A]        ⏸
```

- **생명**: 하트 5개(잃으면 빈 하트 ♡로 — 개수+모양 이중 표현).
- **점수**: 천 단위 콤마. 점수 상승 시 숫자만 잠깐 굵게(카운트업 애니메이션 금지 — 시선 분산).
- **콤보**: ×N 배지. 0일 땐 숨김. 5 단위로 배지 크기 1단계↑.
- **레벨**: LV N. 레벨업 순간만 1초 강조.
- **입력 모드 표시등**: `[영문 A]` 초록 / 한글 IME 감지 시 `[한글 ⚠]` 주황 + "한/영 키를 눌러 영문으로 전환!" 배너(§8, 한국 교실 필수 기능).
- **일시정지 버튼**(마우스용). 키보드는 ESC/P.
- CPM·정확도는 **게임 중 표시하지 않는다** — 결과 화면 전용(플레이 중 시선은 블록에만).

### 7.2 결과 화면

- 표시 항목: 총점 / 등급(S·A·B·C·D) / 정확도(정타 ÷ 총 타건, %) / **CPM**(분당 정타 수 — 코드 타이핑엔 WPM보다 CPM이 정직. WPM 병기: CPM÷5) / 최대 콤보 / 처치 블록 수 / 도달 레벨 / 완주 여부.
- **등급표**: S = 정확도 ≥ 97% 그리고 완주 · A = 정확도 ≥ 93% 그리고 레벨 7+ · B = 정확도 ≥ 85% 또는 레벨 5+ · C = 레벨 3+ · D = 그 외.
- **약점 분석 (학습 피드백 — 이 게임의 차별점)**:
  - "자주 틀린 글자 TOP 3" (예: `:` 7회, `"` 5회, `(` 4회)
  - "자주 틀린 유형" — 오타가 난 스니펫의 category 집계 (예: condition 40%)
- **최고 기록**: localStorage에 난이도별 저장, 갱신 시 "신기록!" 배지.
- **v1.2(2026-07-11) 성장 비교**: 최고 기록과 별개로 <직전 판>(codedrop_last, 난이도별)과의 차이를 결과 화면에 표시 — 점수·CPM·정확도·최대 콤보의 ▲▼(색+기호 이중). 첫 판이면 "첫 기록!" 안내.
- **v1.2 성장 사다리**: 완주 시 다음 난이도 도전 버튼(비기너→초급→중급→고급). 게임 오버 시엔 표시하지 않는다(재도전 유도는 R키).
- **v1.2 온보딩**: 생애 첫 플레이(codedrop_seen 부재)에만 "첫 글자를 치면 블록이 잠겨요" 힌트 배너 — 첫 타겟 잠금 성공 시 영구 종료.
- **결과 요약 복사 (제출용)** 버튼 — 학번·이름 입력칸(결과 화면 상단) + 클립보드 복사:
  ```
  [코드 낙하 결과]
  학번: 10101 / 이름: 홍길동
  난이도: 중급 · 등급: A · 점수: 12,480
  정확도: 94.2% · CPM: 182 · 최대 콤보: 12 · 레벨 7 도달
  일시: 2026-07-08 14:30
  ```
  (교사가 패들렛/구글폼 제출로 수합하는 활동지 패턴 — 기존 특강 자료와 동일 UX.)

## 8. 접근성 (WCAG 2.2 AA / KWCAG 2.2)

1. **대비**: 본문·코드 텍스트 대 배경 ≥ 4.5:1, 블록 테두리·하트 등 UI 그래픽 ≥ 3:1. 최종 팔레트는 TECH_SPEC §6의 값 사용(검증 완료 수치만 채택).
2. **색 비의존 (KWCAG 필수)**:
   - 정타 = 색 변화 **+ 글자 채움(굵기/배경칠)** · 오타 = 색 **+ ✗ 아이콘 + 진동(모션 허용 시)** · 타겟 블록 = 색 **+ ▶ 마커 + 굵은 테두리**.
   - 하트는 채움/빈 모양(♥/♡)으로 개수 병행 표현.
3. **키보드**: 게임 자체가 키보드 전용으로 완결 — 모든 화면 전환이 Enter/ESC/R/화살표로 가능해야 하고, 버튼에는 포커스 링(2px, 대비 3:1↑) 필수. 포커스 순서는 시각 순서와 일치.
4. **모션 배려**: `prefers-reduced-motion` 시 — 블록 진동·소멸 축소 애니메이션·레벨업 연출을 전부 무모션 대체(테두리 점멸·즉시 제거)로 전환. 낙하 자체는 게임 본질이라 유지하되 **낙하가 부담스러운 학생용으로 초급 레벨 1을 안내**.
5. **사운드**: 기본 켜짐이되 첫 화면에 음소거 토글(설정 저장). 소리는 정보의 유일한 전달 수단이 되면 안 됨 — 모든 사운드 피드백에 시각 대응물 존재(위 2번).
6. **텍스트 크기**: 설정에서 코드 글자 크기 3단계(16/19/22px). 200% 브라우저 확대에서 레이아웃 파손 없음.
7. **한/영 IME**: 한글 조합 입력 감지 시 즉시 경고 배너(7.1) — 장애가 아니라 한국 교실의 현실적 접근성 이슈.

## 9. 디자인 톤 — "코드 에디터 안에서 벌어지는 게임"

- **컨셉**: VS Code 다크 테마를 닮은 정적인 무대. 배경 짙은 네이비(에디터), 얇은 줄번호 거터를 배경 장식으로. 블록은 에디터의 코드 라인이 떨어지는 느낌.
- **타이포**: 코드는 모노스페이스(D2Coding, Consolas 폴백 — 웹폰트 없음, 오프라인 동작). UI 텍스트는 시스템 산세리프.
- **이펙트 절제 원칙 (구현자 준수)**:
  - 허용: 블록 소멸 0.18초 페이드+축소, 오타 진동 0.15초, 레벨업 텍스트 1초, 콤보 배지 크기 변화.
  - **v1.1 추가(2026-07-10) 허용 3종**: ① 블록이 바닥 72% 지점을 넘으면 **위험 경고**(테두리 주황+점선 — 색+모양 이중, 타겟 중 블록 제외) ② 타건음 피치가 콤보에 따라 계단식 상승(최대 +12단, 시각 자극 없이 리듬 강화) ③ HUD 레벨 옆 **진행 도트 ●●○…**(다음 레벨까지 처치 수 상시 표시).
  - **금지: 화면 흔들림, 파티클 폭발, 배경 번쩍임, 카운트업 숫자 애니메이션, BGM.** 효과음은 타건·처치·오타·낙하 4종의 짧은 신스음뿐(WebAudio 생성, 음원 파일 없음).
  - 근거: 타이핑 게임의 몰입은 시각 자극이 아니라 **입력 리듬**에서 온다. 화면이 조용할수록 타이핑에 집중한다. (그리고 교실 빔프로젝터에서 번쩍임은 민폐다.)
- 색은 기능색 4개만: 기본(회백) / 정타·성공(초록 계열) / 오타·위험(적색 계열) / 타겟·강조(청색 계열). 정확 값은 TECH_SPEC §6.
